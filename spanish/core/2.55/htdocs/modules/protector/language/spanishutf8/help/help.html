Help: Protector

Descripción

Protector es un módulo para defender su CMS XOOPS de varios ataques maliciosos.

Instalar/desinstalar

Primero, defina XOOPS_TRUST_PATH en mainfile.php si nunca lo ha hecho.

Copie html/modules/protector en el archivo dentro de su XOOPS_ROOT_PATH/modules/
Copie xoops_trust_path/modules/protector en el archivo dentro de su XOOPS_TRUST_PATH/modules/

Ponga escribible los permisos de XOOPS_TRUST_PATH/modules/protector/configs

= Como rescatar =


Si usted ha sido baneado de Protector, elimine los archivos bajo XOOPS_TRUST_PATH/modules/protector/configs/

Introducción para filter-plugins en este archivo.

- postcommon_post_deny_by_rbl.php
un plugin anti-SPAM.
Todos los Mensajes desde la IP registrada en RBL serán rechazadas.
Este plugin puede disminuir el rendimiento de Mensajes, en especial de chat módulos.

- postcommon_post_deny_by_httpbl.php
un plugin anti-SPAM.
Todos los mensajes desde IP registrada en http:BL will serán rechazadas.
Antes de usar esto, optenga HTTPBL_KEY de http://www.projecthoneypot.org/ y póngalo dentro de el archivo filtro.
define( 'PROTECTOR_HTTPBL_KEY' , '............' ) ;

- postcommon_post_need_multibyte.php
un plugin anti-SPAM.
Mensajes sin caracteres multi-byte serán rechazados.
Este plugin es solo para sitios en japanese, tchinese, schinese, y korean.

-postcommon_post_htmlpurify4guest.php
Todos los enviós de datos enviados por invitados serán purificados por HTMLPurifier.
Si usted permite a los invitados publicar HTML, Le recomiendo encarecidamente que active esta carácterística.

-postcommon_register_insert_js_check.php
Este plugin previene su sitio de registro de usuarios robots.
Se Require JavaScript trabajando en el navegador de los visitantes.

-bruteforce_overrun_message.php
Especifica un mensaje para visitantes que prueban con contraseñas equivocadas mas de las veces especificadas.
Todos los plugins nombrados *_message.php especifica el mensaje para acceso denegado.

-precommon_bwlimit_errorlog.php
Cuando por desgracia hay limitaciones de ancho de banda, este plugin lo registra en en registro de Apache.

Todos los plugins nombrados *_errorlog.php registro de algunas informaciones en Apaches error_log.
