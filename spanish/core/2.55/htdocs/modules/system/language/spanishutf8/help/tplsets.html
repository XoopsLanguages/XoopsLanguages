Ayuda: Plantillas

Descripción

XOOPS permite una cierta separación entre la presentación actual del sitio (la “piel” o interfaz del usuario) y el código que se ejecuta en en el backend. El primero de ellos es manejado por un sofisticado mecanismo basado en temas gráficos. Un tema puede incluir, en su forma básica, las esenciales plantillas HTML que definen la interfaz visual del sitio, así como las hojas de estilo y las imágenes necesarias para complementar la interfaz visual. Los temas son conmutables en la interfaz de administración, por lo que cambiar toda la apariencia de un sitio XOOPS sólo es cuestión de cambiar el valor de un selector y haciendo clic en "Aceptar". traducción del inglés al español XOOPS también proporciona a los administradores del sitio la posibilidad de permitir a los usuarios elegir entre algunos de los temas disponibles

Generalmente hablando, un theme es el marco de alto nivel visual de un sitio: el tema define, por ejemplo, el diseño general del sitio, por el contrario, no suele definir la distribución de contenido específico dentro de un bloque. Este es manejado por una entidad diferente llamada conjunto de plantillas. Un conjunto de plantillas es un conjunto de plantillas HTML que definen la presentación del contenido dentro de todos los bloques que forman un sitio Web de XOOPS. Si queremos una metáfora más clara de esto, podríamos decir que el tema es como el arquitecto de una casa, mientras que el conjunto de plantillas es como el decorador de interiores. El primero establece las paredes y define la distribución general, pero es la segunda que está a cargo de “vestir” esas paredes.

Si el tema activo es un sencillo que sólo incluye un archivo de tema, los archivos CSS y algunas imágenes, XOOPS utiliza la plantilla predeterminada para definir la presentación de los bloques. Sin embargo, algunos temas complejos son lo suficientemente sofisticados como para incluir su propio conjunto de plantillas. En este caso, el administrador del sitio debe utilizar la interfaz de administración para fijar tanto el tema y el conjunto de plantillas.

Como es el tema magia a cabo? Es decir, ¿cómo pueden las plantillas HTML incluir contenido dinámico? Esto no se hace con espejos y humo, pero si con un motor de plantillas llamadas Smarty que hace la función de lo que solía ser llamado interpolación de variables. Expresado de forma sencilla, podríamos decir que el sistema XOOPS utiliza archivos de PHP para mantener la lógica de los módulos (Acceso a la base de datos, la asignación de variables, gestión de derechos...), archivos HTML con las variables de Smarty insertas para gestionar la interfaz visual real, y el motor Smarty para insertar valores y las cadenas en lugares específicos en el código HTML que crea las páginas que ve el usuario final. Clonando y descargando plantillas

Si desea crear un nuevo conjunto de plantillas puede hacer una copia del conjunto predeterminado (o cualquier otro) mediante la selección del enlace ‘clonar’ (Se le pedirá para dar un nuevo nombre al conjunto). A diferencia de la opción predeterminada, otros juegos de plantilla se pueden editar libremente. También puede guardar una copia de un conjunto de plantillas en el disco mediante la selección del enlace ‘descargar’, o eliminarla con el enlace ‘eliminar’. También es posible el establecimiento de un enlace adicional ‘Generar’ junto a nuevos módulos para que las plantillas no han sido creadas todavía.

Editando plantillas

Puede ver una lista completa de plantillas disponibles navegando por la vista de árbol de los directorios. Cuando se selecciona una plantilla, XOOPS la abrirá en un editor donde se pueda editar.

Durante la edición de plantillas debe habilitar la preferencia que se actualiza automáticamente plantilla de módulo. archivos html si hay nuevos archivos bajo themes/su theme/directorio de plantillas para el tema actual (Módulo System -> Preferencias -> Configuración General -> actualización del módulo de plantilla archivos .html de themes/su theme/directorio de plantillas? -> ‘Si’). Esto debe ser desactivado una vez que el sitio se haga público.

Si accidentalmente daña una plantilla no se olvide que usted puede recuperar una ‘copia buena’ de la plantilla predeterminada. Si se trabaja con plantillas no estandares puede ser buena idea guardar una copia en el disco antes de que empiece a modificarla.
