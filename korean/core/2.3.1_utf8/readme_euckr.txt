##XOOPS 한국어버전 설치법

--빠른 시일내에 
한국어 언어팩이 아닌...
XOOPS 한국어버전을 자체적으로 업할 예정이오니..
그때까진...아래의 방법으로
언어팩을..이용해 한국어버전으로 만드시기 바랍니다.

(1)XOOPS 코어를 다운받는다.
xoops.org 에서 최신버전을 다운받습니다.
http://www.xoops.org

(2)저희 사이트의 다운로드코너나 
혹은.. dev.xoops.org 의 한국어버전 프로젝트 페이지에서
한국어 언어팩을 다운받습니다.
(EUC-KR 과 UTF-8(유니코드) 버전중..선택) 
프로젝트페이지:
http://dev.xoops.org/modules/xfmod/project/?group_id=1189

(3)언어팩을 압축해제하여..
내용물을.. (1)에서 다운받은..코어파일들에..
추가합니다.
(이미지파일의 경우엔..덮어쓰기)

(4)이제.. 설치를 시작합니다.
설치는 설치마법사를 통해..간단히..되므로..
설치마법사를....그냥..따라가시면..됩니다.
--설치시 사용할 언어는..
EUC-KR의 경우.. korean 을
UTF-8 의 경우... koreanutf 를 선택하세요.
--XOOPS 루트 혹은... XOOPS 루트/install/index.php 를 
브라우저로 액세스하시면.. 설치마법사가 시작됩니다.


##XOOPS/XOOPS Cube업그레이드 방법

1, 사전준비
업그레이드전에 반드시 데이타베이스 백업을 해 두시기 바랍니다.(백업의 습관화)
그리고..XOOPS 현 버전의 파일들도 가능하면..업그레이드전에 백업해 두시는 것이 좋습니다.
개인적으로 작업한 HACK(개조)의 내용들을..체크해 둡니다.

2, 업그레이드용 파일을 다운받습니다.

예) xoops-2.0.9_to_2.0.9.2.tar.gz 
( www.xoopscube.org 에서 다운받으시거나...아니면..
..저희 한국어지원사이트에서..한국어 버전용을..준비할 예정이니..
해당 파일을..다운받으시면 됩니다.)

3, 압축을 푸신 후 새버전의 파일들로..이전 버전의 파일을 덮어쓰기하시면..됩니다.
(이때..개별적으로 작업한..HACK들의 내용은..미리..확인해 두셔야겠죠.)

4, 각 모듈별로..파일이 업그레이드되었다면..
관리자메뉴에서..해당 모듈을..업그레이드 해 주셔야만 합니다.

5, 이젠..정상동작하는지 확인하시면..되겠죠

PS:가능하면..테스트서버에서..업그레이드테스트를..해보신후..
문제가 없을..경우..공개서버에..업그레이드를 적용하시길..권합니다.

##문제발생시 기본대처법

1, 테마를 변경하였는데..반영되지 않는다
관리자 메뉴의 일반설정에서...
theme/디렉토리로부터의 자동 업그레이드기능을 사용 이란 항목을..
사용함으로 설정하신 경우엔 자동으로 변경이 반영됩니다만..
(사이트 공개시에는 보통..꺼두는게 좋습니다.)
그렇지 않은 경우는 templates_c 디렉토리에 
PHP화되어 캐쉬되어 있는 파일을..삭제하시면
변경내용을 바탕으로 다시 캐쉬하게 되므로..변경사항이 반영되게 됩니다.

참고>모듈의 템플릿을..변경한 경우엔..모듈업그레이드를 하시면..반영됩니다.

2, 페이지에 공백페이지처럼 아무것도 표시되지 않는다.
이처럼..아무것도 표시되지 않는 경우는 에러가 발생했기 때문입니다.
관리자메뉴의 일반설정에서..디버그모드(PHP 디버그)를 켜신 후 
다시 접속하면..해당 에러메세지가 나타나오니...디버깅해주시기 바랍니다.

3, 로그인화면이 표시되지 않는다.
이 경우엔..먼저.. 
/user.php 에 접속해 보신 후 로그인폼이 표시되면..이를..이용해 로그인하시면.됩니다.
만약.. 정상적으로 표시되지 않는다면
임의로 로그인 폼을..작성하셔서
<html>
<body>
<form action="user.php" method="post">
아이디: <input type="text" name="uname" size="12" value="" /><br />
패스워드: <input type="password" name="pass" size="12" /><br />
<input type="hidden" name="op" value="login" />
<input type="submit" value="로그인" />
</form>
</body>
</html>
저장하신 후 
이 파일을 통해 로그인하시면 됩니다.

4, 관리자페이지로 바로 접속하시려면..
/admin.php 또는
/modules/system/admin.php
로 접속하시면 됩니다.
(물론..로그인한 상태라야겠죠. 3번의 방법으로 먼저..로그인한후..접속하세요)

##인덱스화면 구성에 대하여...

XOOPS에서 첫인덱스화면을 꾸미는 
일반적인 방법은..2가지있습니다.
(물론..HACK 하시면..다양하게..바꾸실 수 있습니다만..쩝)
(1) 시작모듈의 지정없이 블록만으로 구성
(2) 시작모듈을..지정해서..리다이렉트시킴
(시작모듈의 지정은..관리자메뉴-일반설정에서 하실 수 있습니다)
먼저 
(1) 시작모듈의 지정없이 블록만으로 구성하는 경우
일반설정에서 시작모듈을..지정하지 않으면..
첫인덱스화면..접속시..블록만으로 화면을..표시하게 됩니다.
(표시할 블록은.. 각..블록의 설정에서..지정하신..
표시할 곳..즉...Top페이지 혹은..모든페이지를 선택하셨다면..그 블록이 표시됩니다.)
www.xoops.org 의 첫화면이..바로..그런 경우입니다.
커스텀 블록(직접작성한 블록)과 각종 모듈의 블록을..조합해서 
첫화면을..구성한..것입니다.(이 경우..중앙블록이 없으면..안되겠죠^^;;)
그래서.. index.php 를 보시면..
$xoopsOption['show_cblock'] =1;로 중앙블록을..표시하게 하는 겁니다
#페이지구성에 관해선.. 테마토론방의 글을..참조하세요!

(2) 시작모듈을..지정해서..리다이렉트시킴..
현재..XOOPS Korean 사이트에 접속하시면..
포럼모듈로..리다이렉트되는..건..
바로..시작모듈로..포럼모듈을..지정했기 때문입니다.
index.php 의 코드를..참조하세요
