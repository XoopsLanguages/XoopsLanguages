Aide : Modules

Description

Les modules sont des extensions du noyau, ils utilisent les fonctionnalités incluses dans le code du noyau de XOOPS. Ils offrent donc ainsi, la possibilité à l'administrateur du site de fournir aux utilisateurs un contenu dynamique ou statique généré sous une interface Web, ce qui est habituellement pas si évident à mettre en oeuvre.

Une partie de la puissance de XOOPS réside dans sa méthode dont sont gérés les modules.
L'Administrateur peut facilement installer, activer, désactiver ou désinstaller, n'importe quel module (excepté le module système nécessaire nécessaire au noyau).
Il lui est également possible de renommer les modules, et de déterminer, pour l'utilisateur final, l'ordre dans lequel ils apparaîtront dans les menus du site – si vous désirez les faire apparaître bien entendu –, car il vous est également possible de fournir des fonctionalités masquées. Par exemple en définissant les modules actifs qui n'auront pas de présence visible pour l'utilisateur lambda.
Vous pouvez rendre un module invisible un module dans le menu principal en le décochant dans la colonne correspondante.

Pour modifier l'ordre d'affichage des modules (qui sera également celui dans les menus), glissez-déposez simplement les modules à l'emplacement désiré.

Le choix des modules se fera en fonction de ses caractérisques et de la définition de votre besoin pour la réalisation de votre site.
La mise en oeuvre d'un module requiert donc un minimum de planification, à la fois pour son contenu (catégorie, sous-catégorie, etc ...) mais également pour sa disposition visuelle sur le site et les permissions des différents groupes d'utilisateurs. Une astuce est d'aller visiter d'autres sites sous XOOPS afin de visualiser les modules utilisés pour les contenus souhaités.
Vous pouvez survolez les liens sur ces sites avec votre souris, afin d'afficher dans la barre de status de votre navigateur internet, vous devriez voir des adresses comme celle-ci : http://le_site.com/modules/news/....
Tous les modules étant stockés dans le répertoire /modules/, l'élément suivant dans l'adresse ("news" dans notre exemple) donne le nom du module qui affichera le contenu principal de la page de ce lien.

En cliquant sur ce lien,vous pourrez visualiser un exemple de mise en action du module. Une navigation sur d'autres sites sous XOOPS vous donnera une idée plus précise des capacités de présentation et d'organisation du contenu.
