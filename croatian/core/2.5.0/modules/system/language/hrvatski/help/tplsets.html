Pomoć: Predlošci

Opis

XOOPS omogućava nam odvajanje između stvarne prezentacije na stranici (“skina” ili korisničkog sučelja) i koda koji se vrti u pozadini. Prvim se rukovodi sofisticiranim mehanizmom baziranom na globalnoj temi. Tema može uključivati, u svojem osnovnom obliku, važne HTML predloške koji definiraju vizualno sučelje stranice, kao i stilove i slike potrebne za kompletno vizualno sučelje. Teme su izmjenjive preko administracijskog sučelja, pa se mijenjanje cijelog izgleda XOOPS portala svodi na obični klik na “Prihvati”. XOOPS također pruža administratorima mogućnost da korisnici sami biraju između nekoliko dostupnih im tema.

Prosto rečeno, tema je vizualni framework stranica na visokoj razini: tema definira, recimo, opći izgled stranice; s druge strane, obično ne utjeće na distribuciju sadržaja unutar bloka. To se definira drugim subjektom zvani set predložaka. Set predložaka je skup HTML predložaka koji definiraju prezentaciju sadržaja unutar svih blokova koji čine XOOPS web strenice. Ako želimo još jednostavniju metaforu, mogli bismo reći da je tema arhitekt kuće, dok je set predložaka dekorater unutrašnjosti. Prvi dio postavlja zidove i definira općenitu distribuciju, dok je drugi zadužen za “površinsku obradu” tih zidova.

Ako je aktivna tema jednostavna koja uključuje samo datoteku, CSS datoteku i neke slike, XOOPS zapošljava zadani predložak da definira prezentaciju blokova. Kako bilo, neke dovoljno složene teme mogu imati svoje vlastite predloške. U tom slučaju, administrator stranica mora koristiti administracijsko sučelje kako bi podesio oboje, temu i set predložaka

Kako se postiže magija teme? Tako je, Kako HTML predlošci mogu uključivati dinamički sadržaj? To se ne postiže zrcalima i dimom, nego mudrim engine-om predložaka zvanim Smarty koji čini djelovanje koje smo navikli nazivati interpolacija varijabli. Jednostavno rečeno, mogli bismo reći da XOOPS sustav koristi PHP datoteke koje sadrže logiku modula (pristup bazi podataka, podešavanje varijabli, upravljanje pravima...), HTML datoteke sa ubačenim Smarty varijablama kako bi stvarno vizualno sučelje i Smarty engine koji ubacuje vrijednosti i tekstove na određena mjesta unutar HTML koji tvori stranice koje vidi krajnji korisnik. Sve u svemu, što to vama znači, možemo objediniti: zahvaljujući sustavu tema, XOOPS dozvoljava dizajnerima izradu još zanimljivijih i vizualno dorađenijih web stranica bez previše brige o tome što se dešava ispod haube.

Kloniranje i download setova predložaka

Ako želite stvoriti korisniči set predložaka možete napraviti kopiju zadanog seta (ili bilo kojeg drugog) izabirom ‘kloniraj’ linka (Morat ćete upisati novo ime za set). Za razliku od zadanog, drugi setovi predložaka se mogu slobodno uređivati. Također možete spremiti kopiju seta predložaka na disk izabirom ‘download’ linka, ili ga ukloniti sa ‘izbriši’ linkom. Ponekad možete vidjeti dodatni ‘Stvori’ link odmah do novih modula za koje predlošci još nisu stvoreni.

Uređivanje predložaka

Možete vidjeti cijelu listu dostupnih predložaka pregledavanjem stablastog prikaza mape. Kad izaberete predložak, XOOPS će otvoriti jedan uređivač gdje ga možete urediti.

Dok uređujete predloške morate uključiti opciju u postavkama koja automatski ažurira predloške modula .html daoteke ako postoje novije datoteke u themes/your theme/templates mapi za trenutnu temu (Sustavski modul -> Postavke -> Opće postavke -> Ažuriraj predložak modula .html datoteke iz vašeg themes/your theme/templates mape? -> ‘Da’). Ovo bi trebalo biti ugašeno kad stranice postanu dostupne javnosti.

Ako slučajno zabrljate predložak ne zaboravite da možete povratiti ‘dobru’ kopiju iz zadanog seta predložaka. Ako radite sa nestandarnim predllošcima dobra ideja bi bila pohraniti ih na disk prije nego ih krenete mijenjati.
