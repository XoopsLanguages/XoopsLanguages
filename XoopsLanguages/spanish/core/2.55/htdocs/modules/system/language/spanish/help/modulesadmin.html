Ayuda: Módulos

Descripción

Los módulos son el software de construcctor que, aprovechando la infraestructura que proporciona XOOPS, permite al admin ofrecer a los usuarios del sitio, contenido dinámico o estático generado mediante una interfaz Web que a menudo no es muy fácil de usar.

Parte del poder de XOOPS reside en la forma en que los módulos son gestionados. El Admin facilmente puede instalar, desinstalar, activar o desactivar algún módulo específico (excepto el módulo System, por razones ovias). También es posible renombrar los módulos, y determinar el orden en que cada uno aparecerá para el usuario final en el menú del sitio – si usted desea que aparezcan todos–, porque tambien puede proporcionar una funcionalidad oculta, la definición de módulos activos que no tengan presencia visible para el usuario ocasional. Por ejemplo, se podría definir un módulo como no visible en el menú mediante la desactivación en el icono de verificación en la columna del menú. Así para el módulo de Noticias, podría enviar un mensaje privado a un grupo selecto de sus usuarios y decirles que para acceder a ese directorio tecleando la dirección en el campo de dirección URL, algo así como http://www.susitio.com/modules/news.

Para cambiar el orden de los módulos (que se verá reflejado en el menú), sólo tiene que arrastrar y soltar los módulos a la ubicación deseada.

La instalación de un módulo necesita un poco de planificación, tanto para los módulos visibles y también el diseño de las páginas. Es una buena idea revisar otros sitios XOOPSs y ver qué módulos se utilizan para mostrar los distintos tipos de contenido. Esto se hace fácilmente mediante la apertura de cualquier sitio XOOPS y se ubica sobre los enlaces del menú de los paneles laterales izquierdo y derecho. Al mismo tiempo ver el aspecto en que aparecen las URL en la barra de estado del navegador. Debería ver las direcciones de esta manera: http://el_sitio.com/modules/wfsection/.... Todos los módulos son almacenados en el directorio /modules/, por lo tanto el sector siguiente de la dirección (en este ejemplo 'wfsection') muestra el módulo que mostrará el contenido de ese enlace del menú.

Si abre la página del enlace mostrará un ejemplo de cómo se utiliza el módulo y el tipo de contenido que muestra. Revisando otros sitios como este también le dará una buena perspectiva de las capacidades de distribución y el diseño.
