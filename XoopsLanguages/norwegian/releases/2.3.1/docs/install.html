Installing XOOPS 


First time installation


Preface:

XOOPS is an open-source Object-Oriented Web publishing system written in PHP. It is an ideal tool for developing small to large dynamic community websites, intra company portals, corporate portals, weblogs and much more.
XOOPS is released under the terms of the GNU General Public License (GPL) and is free to use and modify. It is free to redistribute as long as you abide by the distribution terms of the GPL.

To install XOOPS for the first time, you'll need to have the minimum following server software pre-installed:
* WWW Server (Apache, IIS, Roxen, etc)
* PHP 4.3.0 or higher (5.2 or higher recommended)
* MySQL 3.23 or higher (4.1 or higher recommended)

Before starting the install, be sure to have:
* Setup WWW server, PHP and database server properly.
* Prepare a database for your XOOPS site.
* Prepare user account and grant the user the access to the database.
* Make the directories of uploads/, cache/ and templates_c/ and the files of mainfile.php writable.
* If you need to install protector module from GIJOE, replace the file mainfile.dist.php with /extras/mainfile.dist.php.protector (remember to rename the file to htdocs/mainfile.dist.php).
* For security considerations, you are encouraged to move directories xoops_lib (for XOOPS libraries) and xoops_data (for XOOPS data) out of document root, or even change the folder names.
* Make the directory of xoops_data/ writable; Create and make the directories of xoops_data/configs/, xoops_data/caches/, xoops_data/caches/xoops_cache/, xoops_data/caches/smarty_cache/ and xoops_data/caches/smarty_compile/ writable.
* Turn cookie and JavaScript of your browser on.


Installing locally

If you are running a local environment for development or testing, make sure that you have the previous requirements met. Once this is done, copy the contents of the htdocs directory (from the XOOPS distribution file or SVN) to the root document path of your web environment. Once the files are copied there, you can start the install by typing "http://yoursite.com". This will start the install process.

Installing on a hosted platform

If you are running in a hosted environment, unpack the XOOPS files locally or on the server if you have telnet or SSH access. Once you done this, make sure to move or copy all XOOPS files from the htdocs directory to your root web directory (your provider usually provides this location with directions). Once the files are copied there, you can start the install by typing "http://yoursite.com". This will start the install process.

Continuing the install

Just follow step-by-step instructions in installer.

  Installasjon av XOOPS


 

Første gangs installasjon


Forord:

XOOPS er et omfattende, OO (Objekt Orientert), dynamisk content management system (CMS)  skrevet  i PHP og er lett å bruke. XOOPS er et utmerket verktøy for å utvikle alt fra små til store dynamiske nettsamfunn, intranett portaler, bedriftsportaler, nettblogger og mye mer.
XOOPS utgis under reglene som gjelder for GNU General Public License (GPL) og er gratis å bruke og modifisere. Det er også fritt å videredistribuere så lenge reglene i GPL følges.

For å installere XOOPS for første gang trenger du minimum server programvare installert:

* WWW Server (Apache, IIS, Roxen, etc)
* PHP 4.3.0 or higher (5.2 eller høyere er anbefalt)
* MySQL 3.23 or higher (4.1 eller høyere er anbefalt)


Før du  begynner installeringen sørg for at følgende er utført:

* Sørg for at WWW server, PHP og database server er satt opp skikkelig.
* Opprett eller gjør klar en database for din XOOPS side.
* Gjør klar en en brukerkonto og gi denne adgang til databasen.
* Gjør mappene uploads/, cache/ and templates_c/ og filen mainfile.php skrivbare.
* Om du trenger å installere protector modulen fra GIJOE, (anbefales!) bytt filen mainfile.dist.php med  /extras/mainfile.dist.php.protector (husk å endre filnavnet til htdocs/mainfile.dist.php).
* For sikkerhetsgrunner anbefales det at du flytter mappene xoops_lib (for XOOPS biblioteker) og xoops_data (for XOOPS data) ut av roten av ditt websted eller endrer mappenavnene.
* Gjør mappen xoops_data/ writable; Opprett og gjør mappene xoops_data/configs/, xoops_data/caches/, xoops_data/caches/xoops_cache/, xoops_data/caches/smarty_cache/ and xoops_data/caches/smarty_compile/ skrivbare.
* Aktiver cookie og JavaScript lesing i din nettleser.



Lokal installasjon

Om du skal installere lokalt for testing eller utvikling, sørg for at du har ovenstående krav installert. Kopier innholdet av htdocs mappen til roten av webområdet. Etter at filene er kopiert, kan du starte installasjonen ved å skrive "http://yoursite.com".  Dette vil starte installasjonsprosessen.

Installasjon hos en utbyder (hosted server)

Om du skal installere til en server hos en utbyder, pakk ut XOOPS filene lokalt eller på serveren om du har telnet eller SSH adgang. Når du har gjort dette, flytt eller kopier alle filene i mappen htdocs til roten av ditt websted (din utbyder gir deg vanligvis denne informasjonen). Etter at filene er kopiert, kan du starte installasjonen ved å skrive "http://yoursite.com".  Dette vil starte installasjonsprosessen.

  Installation af XOOPS


 

Første gangs installation


Forord:

XOOPS er et open-source objektorienteret Webudgivelse system skrevet i PHP. Det er et ideelt redskab til at udvikle alt fra små til store dynamiske websteder, interne selskabsportaler, virksomheders hjemmesider, weblogs og meget mere..
XOOPS er udgivet under betingelserne i GNU General Public License (GPL) og er gratis at bruge og modificere. Det er gratis at distribuerer, så længe du overholder betingelserne i GPL.

For at installerer XOOPS første gang har du behov for følgde præinstalleret software:

* WWW Server (Apache, IIS, Roxen, etc)
* PHP 4.3.0 eller højere (5.2 eller højere anbefales)
* MySQL 3.23 eller højere (4.1 eller højere anbefales)


Før du begynder installationen, skal du sikre at du har følgende:

* WWW server, PHP og database server.
* Forbered en database til din XOOPS side.
* Forbered database bruger og tildel denne rettigheder til databasen.
* Gør følgende mapper skrivbare uploads/, cache/ og templates_c/ og denne fil skal gøre skrivbar mainfile.php writable.
* Hvis du vil installerer protector modulet fra GIJOE (anbefales!), erstat filen mainfile.dist.php med filen /extras/mainfile.dist.php.protector (husk at omdøbe den bagefter til htdocs/mainfile.dist.php).
* Af sikkerhedsmæssige årsager anbefales det at flytte mapperne xoops_lib (til XOOPS libraries) og xoops_data (til XOOPS data) væk fra root mappen, og/eller omdøbe mappenavnene.
* Gør xoops_data/ skrivbar; Opret og gør følgende mapper skrivbare xoops_data/configs/, xoops_data/caches/, xoops_data/caches/xoops_cache/, xoops_data/caches/smarty_cache/ og xoops_data/caches/smarty_compile/.
* Slå cookie og JavaScript til i din browser.



Lokal installation

Hvis du installerer lokalt på en maskine for udvikling og test, skal ovenstående følges. Når dette er gjort, kan du kopierer indholdet af htdocs mappen (fra XOOPS downloadet ) til rod-mappen. Når filerne er kopieret, kan du starte installationen ved at indtaste "http://dinside". Dette vil starte installationsprocessen.


Installation hos en udbyder

Hvis du installerer hos en udbyder, skal du pakke XOOPS filen ud lokalt eller på serveren, hvis udbyderen tillader det. Når du har gjort det, skal du kopierer filerne i htdocs mappen til rodmappen hos din udbyder (din udbyder fortæller dig oftest hvor denne mappe er.). Når filerne er kopieret, kan du starte installationen ved at indtaste "http://dinside". Dette vil starte installationsprocessen.
